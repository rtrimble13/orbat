As the project develops, this directory will include:

- **API Reference** - Detailed API documentation for all public interfaces
- **User Guide** - How to use the orbat library and CLI tool
- **Algorithm Guide** - Explanation of portfolio optimization algorithms
- **Examples and Tutorials** - Step-by-step guides for common tasks
- **Architecture** - High-level design and architecture decisions

## Contributing to Documentation

When contributing:
- Follow the same style guide as code (see [../CONTRIBUTING.md](../CONTRIBUTING.md))
- Keep documentation in sync with code changes
- Add examples to illustrate concepts
- Use clear, concise language
- Check for spelling and grammar

## Documentation Standards

- Use Markdown (`.md`) for most documentation
- Maximum line length: 120 characters for readability
- Include code examples where appropriate
- Link between related documents
- Keep a consistent structure and tone
